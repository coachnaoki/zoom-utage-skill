# デザイン/文体のカスタマイズ

このスキルはデフォルトで「落ち着いた配色・和文タイポ優先・可読性重視」のレッスンページを生成する。ターゲット層に合わせて色・フォント・Gemini プロンプトを調整できる。

## どこを触るか

`create_lesson.py` の上部にある `CUSTOMIZE` セクション:

```python
# ========================================================================
# CUSTOMIZE: ここから下はあなたのレッスンのターゲット層に合わせて調整する
# ========================================================================

# --- デザイントークン ---
FONT = '"Hiragino Kaku Gothic ProN", "Noto Sans JP", ...'
INK = '#0a1a2e'          # 見出し・強調テキスト
INK2 = '#3a4a5a'         # 本文
GRAY_TEXT = '#5a656b'    # キャプション・ラベル
BG_SOFT = '#f5f8fa'      # カード背景（薄色）
BORDER = '#d4dde3'       # 枠線
ACCENT = '#1e4a7a'       # ボタン・バッジ・アクセント

# --- レッスンの性格（Gemini プロンプトで使う）---
LESSON_CONTEXT = 'オンライン勉強会'
```

## Claude Code に頼む（おすすめ）

Claude Code に話しかけるだけで自動で書き換えてくれる:

> このスキルのターゲット像は「40代・英語コーチング受講生」です。
> `create_lesson.py` の CUSTOMIZE セクション（デザイントークンと LESSON_CONTEXT）
> を、そのターゲットが落ち着いて読める配色・文体に調整してください。

Claude Code がやってくれること:

1. ターゲット像から「どんなトーンが適切か」を判断（若年層ならPOP、40代以上なら落ち着き、等）
2. 適切なカラーパレットを提案して `INK / BG_SOFT / BORDER / ACCENT` を差し替える
3. `LESSON_CONTEXT` をターゲットに合う文脈に書き換える（例「英語コーチング」「経営者向けマーケティング講座」）
4. 必要ならフォントチェーンも書き換える

## 手動で触る場合のヒント

### カラーパレット設計
- **INK（見出し）**: ほぼ黒〜濃紺。純黒（#000000）は避け、#0a1a2e のような暗色が目に優しい
- **INK2（本文）**: INK より少し明るい。本文は INK2 で読みやすく
- **BG_SOFT**: チャプターボックス背景。純白に対して僅かに色のついた薄色（#f5f8fa, #fafaf5 等）
- **BORDER**: カード枠。薄いグレー（#d4dde3, #e0e0d8 等）
- **ACCENT**: ブランドカラー1色。ボタン・バッジに使う

### 禁止事項（のっぺりダサくなる原因）
- 純白 + アクセント1色だけ → グレー階調（100/200/300...）を必ず挟む
- グラデーション背景 → フラット配色の方が大人向けには読みやすい
- 純黒（#000）本文 → 目が疲れる

### LESSON_CONTEXT
Gemini のタイトル・扱った内容・チャプター生成プロンプトに `{LESSON_CONTEXT}` として埋め込まれる。

- テニス → `'テニス勉強会'`
- 英語コーチング → `'英語コーチングセッション'`
- マーケ講座 → `'マーケティング実践講座'`
- 一般 → `'オンライン勉強会'`

プロンプトに入れると、Gemini がその文脈に合わせた語彙でタイトルを作る（例「フォアハンドの打点」vs「TOEIC Part 5 の解き方」）。

## さらに踏み込むなら

`build_rich_elements()` 関数の中身（HTML の構造そのもの）を書き換えればレイアウトも変えられる。カードをグリッド2カラムにしたい、チャプターをアコーディオンにしたい、等。

その時も Claude Code に頼むのが早い:

> `build_rich_elements` のチャプター部分を、クリックで折りたためるアコーディオン形式に変更して。ただし UTAGE の iframe 動画プレイヤーとのチャプタージャンプ機能（`relation_elements`/`relation_id`）は絶対に壊さないで。
